# docs/archived/

Historical long-form documents — design plans, audits, incident postmortems, and migration reports — that describe work which has since shipped. Kept for archaeological context, **not** as authoritative status.

For active or partial-status documents, see `docs/` (root). For local tickets and notes, see `docs/project_incoming/` and `docs/project_archived/`.

This directory is a `experimental/matt`-branch convention; the archived/incoming split is not intended to be proposed upstream as-is.
